Introducción rápida a MVC

¿Qué es el modelo MVC?

MVC (Model–View–Controller) es un patrón de arquitectura de software que separa una aplicación en tres componentes principales:

  • Modelo (Model): maneja los datos y la lógica de negocio.
  • Vista (View): se encarga de la parte visual, lo que ve el usuario.
  • Controlador (Controller): recibe las peticiones del usuario, coordina el flujo y comunica el Modelo con la Vista.

La idea es organizar mejor el código, hacerlo más fácil de mantener y permitir que distintas partes del sistema evolucionen sin afectar todo el proyecto.

Responsabilidades de cada parte

  • Modelo:
    • Acceso a la base de datos.
    • Reglas de negocio (cálculos, validaciones).
    • Representa entidades como Producto, Usuario, Pedido, etc.
  • Vista:
    • HTML, CSS, JS.
    • Muestra la información al usuario (tablas, tarjetas de productos, formularios).
  • Controlador:
    • Recibe la solicitud (por ejemplo: /productos/listar).
    • Pide los datos al Modelo.
    • Selecciona la Vista adecuada y le pasa los datos.

Ventajas de usar MVC

  • Código más organizado y modular.
  • Facilita el mantenimiento y la escalabilidad.
  • Permite que varios desarrolladores trabajen en paralelo (uno en vistas, otro en modelos, etc.).
  • Reutilización de componentes (mismos modelos con distintas vistas).

Flujo básico de MVC

1. El usuario hace una petición (por ejemplo, entra a /productos).
2. El Controlador de productos recibe la petición.
3. El Controlador llama al Modelo para obtener los datos (lista de productos).
4. El Modelo consulta la base de datos y devuelve los datos al Controlador.
5. El Controlador envía esos datos a la Vista correspondiente.
6. La Vista genera el HTML final que el usuario ve en su navegador.

Ejemplo sencillo de MVC con productos DMStore

Supongamos que tienes un módulo para mostrar el catálogo de productos de DMStore usando MVC con PHP (ejemplo simplificado). 

// Modelo: Models/Producto.php
class Producto {
  public static function obtenerTodos() {
    // Ejemplo simple sin BD real:
    return [
      [
        "id"       => "P001",
        "nombre"   => "Perfume DM Intense",
        "categoria"=> "Perfumería",
        "precio"   => 3500.00
      ],
      [
        "id"       => "T010",
        "nombre"   => "Audífonos Bluetooth Pro",
        "categoria"=> "Tecnología",
        "precio"   => 4200.00
      ]
    ];
  }
}
// Controlador: Controllers/ProductoController.php
require_once "Models/Producto.php";

class ProductoController {
  public function listar() {
    $productos = Producto::obtenerTodos();   // Llama al Modelo
    include "Views/productos_listar.php";    // Llama a la Vista y le pasa $productos
  }
}
<!-- Vista: Views/productos_listar.php -->
<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <title>Catálogo DMStore</title>
</head>
<body>
  <h1>Catálogo de Productos DMStore</h1>
  <table border="1">
    <tr>
      <th>ID</th>
      <th>Nombre</th>
      <th>Categoría</th>
      <th>Precio</th>
    </tr>
    <?php foreach ($productos as $p): ?>
      <tr>
        <td><?= $p["id"]; ?></td>
        <td><?= $p["nombre"]; ?></td>
        <td><?= $p["categoria"]; ?></td>
        <td>RD$ <?= number_format($p["precio"], 2); ?></td>
      </tr>
    <?php endforeach; ?>
  </table>
</body>
</html>

En este ejemplo:

  • Modelo (Producto): devuelve la lista de productos.
  • Controlador (ProductoController): coordina la petición, llama al modelo y carga la vista.
  • Vista (productos_listar.php): muestra los datos en una tabla HTML.

¿Por qué usar MVC en proyectos como DMStore?

Para proyectos que crecen (más módulos, más pantallas, más lógica), el patrón MVC ayuda a:

  • Mantener el código limpio y separado por responsabilidades.
  • Agregar nuevas funcionalidades sin romper lo que ya funciona.
  • Reutilizar componentes: por ejemplo, el Modelo de producto se puede usar para web, API, o panel de admin.
Tip: Tu proyecto DMStore ya puede ir adoptando estructura MVC creando carpetas como Models/, Views/ y Controllers/, y definiendo rutas que apunten a cada controlador.
