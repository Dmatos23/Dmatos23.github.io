Introducción rápida a JSON

¿Qué es JSON?

JSON (JavaScript Object Notation) es un formato de texto ligero para el intercambio de datos. Es fácil de leer y escribir para las personas, y fácil de interpretar y generar para las máquinas.

JSON se utiliza ampliamente en aplicaciones web y móviles para enviar y recibir datos entre el cliente (navegador/app) y el servidor, reemplazando en muchos casos el uso de XML por ser más simple y compacto.

Características principales de JSON

  • Basado en una estructura de pares clave–valor.
  • Usa una sintaxis muy similar a los objetos de JavaScript.
  • Es independiente de lenguaje, pero compatible con casi todos (PHP, C#, Java, Python, etc.).
  • Ideal para APIs REST y comunicación cliente–servidor.

Usos comunes de JSON

  • Respuesta de APIs (REST) para aplicaciones web y móviles.
  • Almacenamiento de configuraciones y preferencias.
  • Intercambio de datos entre servicios y microservicios.
  • Definición de menús, catálogos, listas de productos, etc.

Ventajas y desventajas

  • Ventajas: sintaxis simple, menos texto que XML, muy rápido de procesar, estándar de facto en APIs modernas.
  • Desventajas: no soporta comentarios de forma nativa, no tiene esquema formal tan extendido como XSD (XML), menos adecuado para algunos escenarios muy complejos.

Ejemplo sencillo de JSON

A continuación se muestra un ejemplo de JSON donde se define un catálogo de productos (similar a productos de DMStore). Este JSON podría ser la respuesta de un API: 

{
  "catalogo": [
    {
      "id": "P001",
      "nombre": "Perfume DM Intense",
      "categoria": "Perfumería",
      "precio": 3500.00,
      "stock": 25,
      "destacado": true
    },
    {
      "id": "T010",
      "nombre": "Audífonos Bluetooth Pro",
      "categoria": "Tecnología",
      "precio": 4200.00,
      "stock": 10,
      "destacado": false
    }
  ]
}

En este ejemplo:

  • "catalogo" es una propiedad que contiene un arreglo (lista) de productos.
  • Cada producto es un objeto con sus propiedades: id, nombre, categoria, precio, stock y destacado.
  • Este JSON puede ser consumido por JavaScript, PHP u otro lenguaje para mostrar los productos en la web.

JSON vs XML (rápido)

Aunque JSON y XML se usan para intercambiar datos, JSON suele preferirse en aplicaciones modernas por:

  • Ser más compacto y fácil de leer.
  • Tener integración directa con JavaScript en el navegador.
  • Ser el formato por defecto en muchas APIs REST.
Tip: En tu proyecto DMStore puedes usar JSON para definir menús, catálogos de productos y configuraciones, y luego consumirlo con JavaScript o PHP para mostrar la información en pantalla con el estilo visual de tu sitio.
